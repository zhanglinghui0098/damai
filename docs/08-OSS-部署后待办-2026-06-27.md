## 🟡 2. Bucket 防盗链 (Referer 白名单)

**为什么**: 关 BPA 后 Bucket 公共读,理论上任何人 GET 都能拉你的图。防盗链限制:只有从 `damai.net.cn` 域名访问时才返回图,直接访问或别的网站嵌入 → 403。省流量 + 防盗用。

**预计耗时**: 5 分钟

### 2.1 打开 OSS Bucket 防盗链设置
1. 浏览器打开: **https://oss.console.aliyun.com/bucket/oss-cn-hangzhou/damai-zlh-prod/overview**
2. 左侧深色导航栏 → 找到 **"数据安全"** 分类(在 **"文件管理"** 下面, **"数据处理"** 上面)
3. 展开 **"数据安全"** → 点 **"防盗链"**

### 2.2 设置白名单
1. 看到 **"Referer 白名单"** 卡片(中间偏左)
2. 卡片右上角 → 点 **"设置"** 链接
3. 弹出输入框:
   - **Referer 白名单**(支持多个,换行或逗号分隔):
     ```
     https://damai.net.cn
     https://www.damai.net.cn
     http://localhost:3000
     ```
   - **是否允许空 Referer**: **关闭** (默认是关闭,保持)
     - ⚠️ 开了的话,直接浏览器地址栏访问或 cURL 都能下到图(防盗失效)
     - 关闭的话,只有带 Referer 头的请求能下(浏览器 <img> 访问会带 Referer)
4. 点 **"保存"** 按钮(右下角)
5. 弹框 → **"确定"** → 等待 1-3 秒
6. 页面顶部出现 **绿色"修改成功"** 提示 → 关闭弹框

### 2.3 验证 (我会跑)
我跑 3 个 curl 测试:
```bash
# Test 1: 模拟从 damai.net.cn 访问 (带 Referer) → 应该 200
curl -I -e "https://damai.net.cn" https://damai-zlh-prod.oss-cn-hangzhou.aliyuncs.com/uploads/2026/06/mqvjqs7alaye_upload.jpg

# Test 2: 模拟外部网站嵌入 (带别的 Referer) → 应该 403
curl -I -e "https://evil-site.com" https://damai-zlh-prod.oss-cn-hangzhou.aliyuncs.com/uploads/2026/06/mqvjqs7alaye_upload.jpg

# Test 3: 直接 cURL (空 Referer) → 应该 403
curl -I https://damai-zlh-prod.oss-cn-hangzhou.aliyuncs.com/uploads/2026/06/mqvjqs7alaye_upload.jpg
```

**预期结果**:
- Test 1: HTTP 200 ✅
- Test 2: HTTP 403 ❌ (防盗成功)
- Test 3: HTTP 403 ❌ (防盗成功)

**如果 Test 1 也是 403**:
- 检查 Referer 字符串有没有空格、协议头 (http/https) 是否对
- 检查白名单是不是保存了 (回到 2.2 步骤看白名单是否还在)
- 给我截图,我帮你诊断

**如果 Test 2 还是 200**:
- 白名单没生效,等 30 秒再试 (OSS 防盗链配置有秒级延迟)
- 还是 200 → 截图 + 重新设置

### 2.4 在浏览器验证前端
1. 打开 https://damai.net.cn/case (案例页有图)
2. 看图能正常显示 (说明从 damai.net.cn Referer 通行)
3. 直接在浏览器地址栏打开一个 OSS 图 URL → 应该 403 (防盗生效)

## 🟡 3. NAS_BACKUP_HOST 改真 IP (凌晨 2 点前必做)

**为什么**: 现在 `/opt/damai/.env.local` 里 `NAS_BACKUP_HOST=192.168.1.x` 是占位符,凌晨 2 点 cron 跑 `backup-to-nas.sh` 会 ssh 连不上 → log 失败。不改就一直失败,3.6GB 案例库 + 后续生成的图都没备份。

**预计耗时**: 10 分钟 (含找 NAS IP 时间)

### 3.1 找 NAS 内网 IP
1. 找一台跟你 NAS 在**同一个局域网**的设备(家里电脑 / 手机)
2. **方法 A (电脑)**:
   - 浏览器打开绿联 NAS 后台: https://ugreennas.local 或 NAS 后台初始地址
   - 登录后 → **控制台** / **系统设置** → 找 **"网络信息"** / **"IP 地址"**
   - 应该是 `192.168.1.xxx` 或 `10.0.0.xxx` 之类
   - 例: `192.168.1.50`
3. **方法 B (手机)**:
   - 装 **"绿联云"** APP
   - 登录 → 选你的 NAS → 设备详情 / 设置
   - 找 **"IP 地址"**
4. **方法 C (路由器)**:
   - 浏览器打开路由器后台: `192.168.1.1` 或 `192.168.0.1`
   - 登录 → **已连接设备** / **DHCP 客户端列表**
   - 找 **"UGREEN NAS"** 或 **"UGOS"** 之类的设备名 → 记下 IP

### 3.2 验证 NAS SSH 可达
1. 打开你电脑的 **终端** (Mac/Linux) 或 **PowerShell** (Windows)
2. 跑:
   ```bash
   ssh 用户名@NAS_IP
   ```
   例: `ssh zhanglh@192.168.1.50`
3. 第一次连会问 **"Are you sure you want to continue connecting (yes/no/[fingerprint])?"** → 输 `yes` 回车
4. 输 NAS 密码 (你日常登 NAS 的密码,不是阿里云密码)
5. 成功登入 → 看到 NAS 终端提示符 → 跑 `exit` 退出
6. 如果失败:
   - 检查用户名/密码
   - 检查 NAS 是否开了 SSH (绿联 NAS 默认开,如果你禁用了要去后台开)
   - 检查 NAS IP 是不是真的 (路由器里可能有多网卡)
   - 给我截图,帮你诊断

### 3.3 准备备份路径
1. 选一个 NAS 上的**有足够空间**的目录(至少 50GB 空闲,因为 OSS 一年的图能有 20-30GB)
2. 推荐: `/volume1/10T/Hermes/projects/damai-backup` (你的 10T 盘)
3. 如果路径不存在,先在 NAS 上跑 `mkdir -p` 创建:
   ```bash
   ssh zhanglh@192.168.1.50 'mkdir -p /volume1/10T/Hermes/projects/damai-backup && ls -la /volume1/10T/Hermes/projects/damai-backup'
   ```
4. 看到目录建好就 OK

### 3.4 把 NAS 信息发给 Hermes
发我以下 3 项:
1. **NAS 内网 IP** (例: `192.168.1.50`)
2. **NAS SSH 用户名** (例: `zhanglh` 或 `root`)
3. **备份路径** (例: `/volume1/10T/Hermes/projects/damai-backup`)

(我不需要 NAS 密码 — 我会用 ECS 生成的 ssh key 配对登入,或者你告诉我密码我用 sshpass 一次)

### 3.5 等 Hermes 改 ECS `.env.local` + 验证
我拿到后做:
1. ssh 到 ECS, 备份 `.env.local` 为 `.env.local.bak-nas-20260627`
2. sed 改 3 行:
   - `NAS_BACKUP_HOST=192.168.1.50`
   - `NAS_BACKUP_USER=zhanglh`
   - `NAS_BACKUP_PATH=/volume1/10T/Hermes/projects/damai-backup`
3. ssh 到 ECS 跑:
   ```bash
   bash -n /opt/damai/scripts/backup-to-nas.sh  # 语法检查
   /opt/damai/scripts/backup-to-nas.sh           # 实际跑一次
   ```
4. 看 `/var/log/damai-backup.log` 有没有 "===== backup done ====="
5. 告诉你"OK,凌晨 2 点会自动跑"

### 3.6 验证 (凌晨 2:05 后)
```bash
ssh root@47.96.128.172 'tail -20 /var/log/damai-backup.log'
```

**预期结果**:
```

[2026-06-28 02:00:01] rsync ... → zhanglh@192.168.1.50:.../canvas-output/
[2026-06-28 02:00:15] ECS 占用: 3.6G
[2026-06-28 02:00:15] ===== backup done =====
```

**如果失败 (常见错误码)**:
| 错误信息 | 原因 | 修法 |
|---|---|---|
| `ERROR: NAS 不可达` | IP 不对 / ssh 没开 / 防火墙挡了 | 回到 3.1 找真 IP,检查 NAS SSH 服务 |
| `Permission denied (publickey,password)` | ssh 用户名/密码错 | 用真用户名/密码重试 |
| `No such file or directory` | 路径不存在 | 回到 3.3 跑 mkdir 创建 |
| `disk quota exceeded` | NAS 配额满了 | 清理 NAS 老文件,或换大点的路径 |

## 🆘 常见卡点 + 解法

### "我找不到那个 tab"
- 阿里云控制台改版过,**3 种版本**:
  - **新版 (2023+)**: 你现在看到的样子,左侧深色导航 + 右侧白色内容
  - **旧版 (2018-2022)**: 顶部 tab 导航
- 99% 情况你在新版,左侧找就对了
- 找不到就把页面截图发我,我帮你指

### "我点了保存但没反应"
- 等 3-5 秒(OSS 配置有 1-3 秒延迟)
- 没反应就**刷新页面重新来**
- 刷新后状态对的话,说明配置已经生效了,只是当时 UI 没回显

### "我禁用了 Key 但忘了新建怎么办"
- 没事,禁用的 Key 可以再启用
- 列表里找到 **"禁用"** 状态的 Key → 点 **"启用"** 链接
- 不会丢 Key 本身的数据

### "我误删了 Key 怎么办"
- 麻烦但能恢复: 重新创建新 Key,然后更新 ECS `.env.local`
- 旧 Key 不能复活(阿里云安全机制)

### "Bucket 名字写错了"
- 创建后**不能改名** (阿里云限制)
- 只能新建一个 Bucket + 改 ECS `ALIYUN_OSS_BUCKET` + 重新上传所有图(老的在老 Bucket 里,访问不到了)
- 重要: 写 Bucket 名时**仔细**核对
