# 视频案例上传指南

## 文件夹结构

```
public/cases/
├── c01-xxx.mp4              ← 视频文件
├── c02-xxx.mp4
├── ...
├── c12-xxx.mp4
├── thumbnails/              ← 封面图 (可选, 不填就用视频第一帧)
│   ├── c01.jpg
│   └── ...
└── cases.json               ← 元数据 (标题/分类/作者)
```

## 命名规范

- 视频文件: `c01-一句话标题.mp4` (按展示顺序编号 01-12)
- 缩略图: `c01.jpg` (跟视频同名, 后缀 .jpg)
- **不要用中文文件名** (浏览器兼容性差)

## 视频要求

- 格式: `.mp4` (H.264 编码最稳)
- 大小: 单个建议 < 50MB (NAS 临时方案, 备案后转 OSS)
- 时长: 建议 10-60 秒 (太长加载慢)
- 比例: 9:16 (抖音/小红书) 或 16:9 (横版), 保持统一

## 填 cases.json

打开 `cases.json`, 把 12 条数据填完:
- `id`: 跟文件名对应 (c1 → c01-xxx.mp4)
- `creator`: 默认填 "即客传媒 · 天禧派" 或你的名字
- `title`: 视频标题 (15 字内)
- `category`: 9 选 1 — 全部 / 品牌广告 / AI品牌激励计划 / 流量破圈 / 信任转化 / 获客引流 / 电商投放 / 人设打造 / 大脉工具箱
- `hue`: 0-360 任意数字 (色块色相, 没视频兜底用)
