## 项目背景

**客户**：张凌辉（杭州天禧派，城北万象城，2000㎡ 软装/家具零售）
**目标**：为家居经销商打造 AI 视频生成工具，专注"老板出镜 + 交付案例"短视频
**商业定位**：AI 武装的家居营销服务公司（不是 SaaS、不是代运营）

## 核心业务场景

经销商老板用工具做抖音/小红书短视频，3 个核心场景：
1. **老板出镜讲解产品**（自己照片 + 自己声音 + 场景图）
2. **客户家交付案例**（客户家照片 + 场景描述 → 多镜头视频）
3. **产品展示 / 节日促销**（产品图 + 模板文案 → 营销视频）

## 技术选型

### 前端
- **Next.js 14** (App Router) + TypeScript
- **Tailwind CSS** — 不用 CSS-in-JS
- **React 18** — hooks
- **shadcn/ui**（后续）— 不用 Material UI / Ant Design

### 后端
- **Next.js API Routes** — 简单场景用这个
- **FastAPI**（后期）— 复杂任务用 Python
- **飞书 Bitable** = 数据层（不用 PostgreSQL / MySQL）

### 部署
- **Vercel** — 网站部署（CI/CD 自动）
- **GitHub** — 代码托管

### AI 模型
- **即梦 AI**（火山方舟）— 性价比之王
- **Seedance 2.0**（火山方舟）— 多镜头控制
- **可灵 AI**（快手开放平台）— 真人出镜
- **Vidu** — 参考图控制

### 不用
- ❌ PostgreSQL（用飞书 Bitable）
- ❌ Redis（前期用不上）
- ❌ Docker（Vercel 自带）
- ❌ 自建模型（用 API）

## 业务约束（AI 写代码时必须遵守）

### 性能
- 视频生成 API 调用可能 30-120 秒，**前端必须异步**（不阻塞、不卡死）
- 一个任务有 task_id，用户可以查状态

### 错误处理
- API 失败 → 自动重试 3 次（指数退避）
- 3 次都失败 → 切备用模型
- 仍失败 → 友好提示用户 + 记录到 Bitable

### 体验
- 用户是 IT 小白，**所有交互必须大白话**
- 任何"超过 15 秒没动静"必须给进度反馈
- 错误信息必须人话（"网络不好，请重试"而不是 "ECONNREFUSED"）

### 安全
- API key 永远在 .env.local，**不提交 git**
- 用户上传的素材加密存储
- 不在日志里打 API key / 用户隐私

## 沟通原则

- **小步快跑**：每改 1 个文件就验证
- **真实环境验证**：curl 一下、浏览器点一下
- **不积累复杂度**：第 1 版能跑就行，不做完美设计
- **出错立刻报**：bug 出现 5 分钟内通知用户

## 不要做的事

- ❌ 不要做"通用视频生成工具"（必须家居垂直）
- ❌ 不要做"代运营"（必须 AI 自动生成 + 经销商自助）
- ❌ 不要做"完美方案"（先跑通，再优化）
- ❌ 不要绑定 1 个视频模型（必须多模型路由）
- ❌ 不要用 PostgreSQL（用飞书 Bitable）
- ❌ 不要 Docker 化（Vercel 自带）
- ❌ 不要写超过 500 行的单个文件（拆）
